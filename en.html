Technical work

UniversalToolchain / Wist

GitHub Docs NuGet

.NET 10 · modular DSL/compiler/runtime framework

A framework for building embeddable domain-specific languages. Wist is the reference language and proving ground used to validate the generic architecture, optimizations and semantic equivalence across execution modes.

  • Designed the Source → Lexer/Parser → AST → Bytecode → AIR → Optimization → Backend → Execution pipeline.
  • Separated frontend-oriented Bytecode from backend-oriented AIR so language modules do not depend on a concrete backend.
  • Implemented an AIR interpreter and compiled typed execution through CIL/DynamicMethod.
  • Replaced backend-name hardcoding in optimizer legality with capability and supported-intrinsic contracts.
  • Protected language semantics with cross-backend parity, module-combination and architecture-guardrail tests.
  • Built a public facade API, CLI, NuGet preview, package smoke tests, GitHub Actions and VitePress documentation.
  • Investigate hot paths with BenchmarkDotNet, generated CIL and assembly.
